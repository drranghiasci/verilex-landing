# VeriLex AI — MVP Scope (v1)

## MVP Goal
Deliver a focused, secure MVP for solo attorneys and small firms that enables onboarding, firm-scoped access, and basic case visibility/management without scope creep.

## In-Scope Features

### A. Firm Onboarding & Access
- Firm intake submission
- Admin approval
- Firm creation
- Email invite + secure acceptance
- Automatic firm membership linking
- Firm-scoped access control

### B. Active Cases
- Firm-scoped case list
- Core case fields:
  - Client name
  - Matter type
  - Status
  - Created date
- Read-only list view
- Minimal empty states

### C. Case Detail (Minimal)
- View case metadata
- Edit basic fields (client name, matter type, status)
- Plain-text internal notes
- Firm-scoped access enforcement

### D. Documents (Basic)
- Upload documents to a case
- List documents per case
- Download/view files
- No AI, no OCR, no tagging beyond filename

### E. Firm Members (Read-Only)
- Admin can view firm members and roles
- No role editing
- No invite management from UI

## Explicitly Out of Scope (Phase 2+)
- AI drafting or summarization
- Client portals
- Billing or invoicing
- Task automation
- Court integrations
- Calendars
- Analytics dashboards
- Intake-to-case automation
- Multi-firm switching
- Advanced permissions
- Notifications
- External integrations

## Definition of “MVP Complete”
- A new firm can onboard and access MyClient
- Users see only their firm’s data
- Cases can be viewed and minimally managed
- Documents can be uploaded and retrieved
- App is stable, demoable, and deployable

This document defines MVP v1. Any feature not listed here is intentionally excluded.
